Crikket reference plan (no code copying)
Last updated: 2026-04-28

Purpose
- Use Crikket as a reference implementation for capture, upload, and reporting UX.
- Rebuild in BetterBugs stack with our own architecture, data models, and code.
- Do not copy code. Use only ideas, flow structure, and UX patterns.

Rules and constraints
- Crikket is AGPL-3.0. Treat this as reference-only. Do not copy source.
- Preserve BetterBugs architecture (Go API + MV3 extension + shared types).
- Prioritize a stable end-to-end capture flow before advanced features.
- Every step below must be testable with a defined input, output, and UI state.

How to use this document
- Each feature block lists: goal, dependencies, Crikket reference files, BetterBugs target areas, and acceptance checks.
- Build order below is strict. Do not start a later step until earlier steps pass their checks.

Glossary
- Capture artifact: screenshot or video file produced by the extension.
- Debugger payload: action/console/network events captured by injected runtime.
- Upload session: server response that returns presigned storage targets.
- Finalize: server call that attaches metadata and returns a share URL.
- Share link: URL to view a bug report without full app context.

Build order (detailed)

Phase 0: Stabilize baseline (must be green before capture work)
0.1 Extension builds without runtime errors
- Output: extension loads in Chrome and popup opens.
- BetterBugs targets: apps/extension/src/manifest.ts, apps/extension/src/popup.
0.2 API boots and serves health endpoint
- Output: local API responds to GET /health (or equivalent).
- BetterBugs targets: apps/api/main.go, apps/api/internal/routes.
0.3 Storage decision recorded (local, S3, or MinIO)
- Output: chosen storage model and env vars documented.
- BetterBugs targets: SETUP.md, apps/api/.env.example.

Definition of done for the "parity baseline"
- A user can capture screenshot or video from the extension.
- Console + network events are attached to the report.
- Upload session -> direct upload -> finalize works end-to-end.
- Report list/detail works in the popup.
- Share link can open a read-only report.
